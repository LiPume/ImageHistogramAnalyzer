# 迭代计划与 TODO

> 状态说明：`未开始`、`进行中`、`待验收`、`已完成`、`阻塞`。  
> 更新规则：每次重要代码变更后更新对应 TODO；阶段结束前逐项核对验收标准。

## TODO 使用规则

- 任务编号保持稳定，提交、测试报告和会议记录均引用编号，例如 `MVP-04`。
- 每个任务开工前填写负责人；完成后勾选并补充对应提交或测试记录。
- `P0` 为阶段主链路，优先完成；`P1` 为本阶段必要完善项；`P2` 为有余力再做的增强项。
- 依赖未完成时不并行修改高冲突文件；需要调整范围时先更新本文件。
- 任务完成定义：代码/文档产物存在、相关检查通过、TODO 状态同步，不以“代码写完”单独判定完成。
