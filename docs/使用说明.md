# MVP 使用与验收说明

## 一、在 Android Studio 中运行

1. 使用 Android Studio 打开项目根目录。
2. 等待 Gradle Sync 完成。
3. 在 Device Manager 创建或启动 API 36（也可使用 API 23 以上）虚拟机。
4. 在顶部设备列表选择该虚拟机。
5. 点击 Run，运行 `app` 配置。
6. 首页出现“图像直方图分析”和“从相册选择图片”即表示启动成功。

如果虚拟机相册没有图片，可把一张 JPG/PNG 直接拖到虚拟机窗口，或通过 Device Explorer 放入 `/sdcard/Pictures/`，再重新打开图片选择器。

## 二、MVP 主流程测试

1. 点击“从相册选择图片”。
2. 在系统 Photo Picker 中选择一张图片并确认。
3. 等待“正在读取并计算直方图”消失。
4. 向下滚动并检查：
   - 图片预览方向和比例正常；
   - 文件名、分辨率、像素数量和 MIME 类型正确；
   - 出现横轴 0、128、255 的白底黑柱直方图；
   - 最高灰度频次大于 0；
   - 图片解码和核心计算分别显示毫秒值；
   - 核心计算低于 300ms 时显示“已达到”提示。
5. 点击“重新选择图片”，选择另一张图，确认结果被完整替换且应用不崩溃。

## 三、建议测试图片

- 普通手机照片；
- 全黑、全白图片；
- 明显偏暗和偏亮图片；
- 横图与竖图；
- 透明 PNG；
- 小图、约 200–400 万像素图片；
- 超过 1600 万像素图片，用于确认安全上限提示。

## 四、命令行验证

```bash
./gradlew test
./gradlew assembleDebug
./gradlew lintDebug
./gradlew assembleDebugAndroidTest
```

连接一个模拟器后执行 Compose 仪器测试：

```bash
adb devices
ANDROID_SERIAL=emulator-5554 ./gradlew connectedDebugAndroidTest
```

如果设备编号不是 `emulator-5554`，替换为 `adb devices` 显示的编号。

## 五、验收记录

请保存以下截图：

1. 首页；
2. 图片预览和分辨率信息；
3. 完整直方图；
4. 核心计算耗时和 300ms 判断；
5. 如有错误，保存错误提示并记录图片类型和尺寸。

正式 300ms 结论应在真机 Release/接近 Release 构建上多轮测试；Debug 模拟器单次结果只能作为开发阶段参考。
