# 开源项目技术借鉴记录

## 一、记录目的

本文件记录课程项目调研过的开源仓库、可借鉴的技术思路以及本项目的自研边界。参考仓库统一放在 `references/open_source/`，该目录已由 `.gitignore` 排除，不属于本项目源码和提交内容。

获取或更新参考仓库：

```bash
./scripts/fetch_references.sh
```

## 二、参考项目：sixo/pixel-proc

- 上游地址：<https://github.com/sixo/pixel-proc>
- 本地目录：`references/open_source/pixel-proc/`
- 上游许可证：Apache License 2.0，以仓库内 `LICENSE` 为准。
- 项目概况：Android 图像直方图示例，包含 Kotlin 实现、C++/NDK 实现、RenderScript 示例和自定义 View/Canvas 绘制。

### 可以参考的内容

- Bitmap 图像处理：观察如何从 Bitmap 获取像素、拆分颜色通道并组织图像处理流程。
- 直方图归一化：理解频次统计后如何将数据映射到可绘制高度。
- Canvas 绘制：参考直方图数据与绘制组件分离、坐标映射和柱形展示思路。
- Kotlin 与 NDK 性能优化：比较托管代码与原生实现的组织方式，理解批量像素访问、循环开销和跨 JNI 边界的成本。
- 测试思路：参考其针对直方图结果建立 Android 测试的方式。

### 不直接采用的内容

- 不复制 `Histogram.kt`、`HistogramView.kt`、`native-lib.cpp`、布局或测试代码。
- 不沿用其旧版 Android 架构、RenderScript 方案或依赖版本；RenderScript 已不适合作为本课程新项目的主优化路线。
- 不因为上游许可证允许使用，就降低课程项目的原创性要求。

## 三、参考项目：billthefarmer/histogram

- 上游地址：<https://github.com/billthefarmer/histogram>
- 本地目录：`references/open_source/histogram/`
- 上游许可证：GNU General Public License v3.0，以仓库内 `LICENSE` 为准。
- 项目概况：较早期的 Android/Java 相机直方图应用，包含相机预览、颜色转换、直方图 View 和横竖屏交互。

### 可以参考的内容

- Android 图像直方图应用结构：理解图像来源、转换处理、直方图 View 与 Activity 之间的职责关系。
- 图片选择/展示方式：参考图像采集或展示流程中的状态衔接，但本项目会改用现代 Activity Result API 和 Compose。
- 直方图展示交互：参考横竖屏布局、显示/隐藏直方图和图像区域交互的产品思路。
- 组件边界：参考 `CameraActivity`、`CameraPreview`、`Converter`、`HistogramView` 的职责拆分，而不是把所有逻辑堆在 Activity 中。

### 不直接采用的内容

- 不复制其 Java、相机、View、资源或构建脚本代码。
- GPL-3.0 对衍生作品和分发有明确义务。本项目仅进行技术调研，不将其源码混入项目，以避免许可证边界和课程原创性风险。
- 当前版本不复制其相机实现；后续 `BASE-09` 会用现代 Activity Result 与 FileProvider 自研“现场拍照并分析”，仍不照搬其旧版交互和工程配置。

## 四、参考项目：awesome-android-agent-skills

- 上游地址：<https://github.com/new-silvermoon/awesome-android-agent-skills>
- 本地目录：`references/open_source/awesome-android-agent-skills/`
- 上游许可证：Apache License 2.0，以仓库内 `LICENSE` 为准。
- 适用判断：适合作为现代 Android Agent 工作流素材，但不适合整库直接安装到本课程项目。

### 采用的思路

- Compose 无状态组件、状态提升和 ViewModel/StateFlow 边界。
- 结构化协程、生命周期感知、任务取消和可测试 Dispatcher。
- 单元测试、UI 测试、可访问性和性能分析检查项。
- 修改前检查、实现后验证、以数据支持性能结论的工作流。

### 删除或改写的默认要求

- 不引入 Hilt、Room、Retrofit、离线优先、Navigation 或多模块 Convention Plugin。
- 不复制上游示例中的 SDK 和依赖版本，继续使用本项目已验证的 Gradle 配置。
- 不默认增加截图测试框架、不可变集合库或 NDK；只有出现真实需求和测量依据时再评估。
- 将通用 Android 建议改写为本项目专用的选图、Bitmap、灰度公式、256×100 Canvas、300ms 和文档同步规则。

项目定制结果位于 `.github/skills/image-histogram-android/`，并通过 `AGENTS.md` 为项目内代理提供入口。

## 五、本项目自研边界

本项目只参考上述仓库的技术思路和工程组织，不直接照搬代码。以下内容必须由团队自行设计、实现、测试和说明：

- 使用指定公式的灰度转换及统一取整规则；
- 0..255 灰度频次统计与 0..100 归一化；
- Baseline 与 Optimized 两种算法及一致性验证；
- 256×100 黑白直方图的 Compose Canvas 展示；
- 图片选择、预览、状态管理和错误处理；
- 性能计时口径、300ms 判断与真机测试；
- 平均灰度、暗亮占比、标准差及质量判断规则；
- Kotlin/Jetpack Compose 界面、中文注释、测试和全部课程文档。

如果后续确需引用开源项目的少量代码，必须先记录来源、许可证、修改范围和使用理由，并由全组确认；在当前计划中默认不引用任何上游源码。
