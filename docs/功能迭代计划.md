# 功能迭代计划

> 项目名称：基于 Android 的移动端图像直方图计算与性能优化系统
> 文档定位：本文用于记录后续增强功能的开发计划、实现边界、阶段目标、验收标准与风险控制。前期已完成的项目选题、需求分析和概要设计不在本文重复展开，本文重点面向功能迭代与开发过程记录。
> 当前策略：先实现离线稳定、可演示、可解释的增强功能；联网 AI API 暂不作为第一版核心依赖。

---

## 一、迭代背景

当前系统已经完成静态图片直方图计算主流程，包括图片选择、预览、两种直方图计算方案、256×100 黑白直方图绘制、性能耗时展示和基础图像质量分析。为了进一步体现直方图在图像分析和拍摄辅助中的实际价值，后续计划围绕“直方图结果如何使用”继续扩展功能。

本轮迭代不再简单增加与主题无关的功能，而是继续围绕图像直方图、图像质量判断、区域分析和实时拍摄辅助展开。目标是将系统从“能够生成直方图”提升为“能够利用直方图辅助用户理解图像质量并优化拍摄效果”。

---

## 二、总体迭代目标

本轮新增三个增强方向：

| 编号 | 功能方向 | 核心目标 | 第一版策略 |
|---|---|---|---|
| F1 | ROI 区域直方图 | 用户自由框选图片局部区域，只分析该区域 | 将 ROI 区域裁剪为新 Bitmap，复用现有分析流程 |
| F2 | 实时相机直方图与智能拍摄建议 | 基于实时直方图判断画面状态，并辅助调节曝光补偿和补光灯 | CameraX + 本地规则建议引擎 |
| F3 | 直方图应用分析增强 | 增加 RGB 三通道指标和自然语言图像质量判断 | 本地规则生成自然语言，不联网 |

本轮功能整体闭环如下：

```text
静态图片 / ROI 局部区域 / 实时相机帧
        ↓
灰度直方图 + RGB 三通道统计
        ↓
亮度、曝光、对比度、偏色分析
        ↓
自然语言质量判断 / 智能拍摄建议
        ↓
自动优化曝光补偿与补光灯 / 用户手动微调
```

---

## 三、开发原则

1. 不破坏现有静态图片分析功能，原有“全图分析”仍为默认路径。
2. 不删除现有两种直方图计算方案，新增功能应尽量复用已有 `domain/histogram` 和 `domain/quality` 能力。
3. 第一版不强制接真实 AI API，不上传原始图片，不依赖网络环境。
4. 自然语言判断采用本地规则生成，保证离线可用、结果可解释、演示稳定。
5. ROI 第一版采用“自由框选 + 裁剪为新 Bitmap + 复用现有流程”的方式，避免大改 Native 算法接口。
6. 实时相机模式只控制曝光补偿和补光灯，不做变焦、对焦、白平衡、ISO、快门等复杂控制。
7. 自动优化后保留用户手动调节能力，形成“智能建议 + 一键优化 + 人工微调”的交互闭环。
8. 新功能逻辑不得写入 `MainActivity` 或 UI Composable 中，应拆分为独立数据类、分析器、控制器和 UI 组件。

## 四、功能一：ROI 区域直方图

### 4.1 功能目标

用户可以在静态图片预览上自由框选一个矩形区域，系统将该区域作为新的分析对象，重新计算灰度直方图、RGB 三通道统计、图像质量判断和性能耗时。

该功能用于解决“全图直方图无法反映局部区域问题”的不足。例如整张图片整体亮度正常，但人物主体区域偏暗，ROI 区域直方图可以更准确地分析主体区域的曝光和对比度情况。

### 4.2 用户交互流程

```text
选择图片
  ↓
显示全图分析结果
  ↓
点击“选择分析区域”
  ↓
进入 ROI 选择模式
  ↓
用户在图片上拖动矩形框
  ↓
点击“确认区域”
  ↓
系统裁剪 ROI Bitmap
  ↓
将 ROI 作为新分析对象重新计算
  ↓
显示 ROI 直方图、RGB 指标和质量判断
  ↓
用户可点击“恢复全图分析”
```

### 4.3 第一版交互要求

| 功能 | 要求 |
|---|---|
| 进入 ROI 模式 | 静态图片分析页增加“选择分析区域”按钮 |
| 框选方式 | 第一版至少支持拖动生成矩形区域 |
| 区域确认 | 提供“确认区域”和“取消选择”按钮 |
| ROI 分析 | 确认后裁剪 ROI Bitmap，并复用现有分析流程 |
| 状态显示 | 显示当前分析对象为“局部区域” |
| 元数据显示 | 显示 ROI 宽高、像素数、占原图比例 |
| 恢复全图 | 提供“恢复全图分析”按钮 |
| 坐标处理 | 处理图片在 Compose 中的缩放、居中和留白映射 |

### 4.4 实现策略

第一版采用裁剪方案：

```kotlin
val roiBitmap = Bitmap.createBitmap(
    originalBitmap,
    roiLeft,
    roiTop,
    roiWidth,
    roiHeight
)
```

裁剪后的 ROI Bitmap 作为新的分析对象进入现有流程：

```text
ROI Bitmap
  ↓
HistogramCalculator
  ↓
HistogramResult
  ↓
ImageQualityAnalyzer
  ↓
RgbHistogramAnalyzer
  ↓
ImageInsightAnalyzer
  ↓
UI 展示
```

### 4.5 建议新增数据结构


### 4.7 验收标准

- 用户能够在图片上框选一个矩形 ROI。
- 确认 ROI 后，系统显示局部区域的直方图和质量判断。
- ROI 分析不影响原图，全图可以恢复。
- ROI 宽高、像素数和占原图比例显示正确。
- ROI 为空、过小或越界时有中文提示。
- 裁剪后的 ROI 分析结果与将该区域作为独立图片分析的结果一致。

## 五、功能二：实时相机直方图与智能拍摄建议模式

### 5.1 功能目标

新增“智能拍摄助手”页面，系统通过 CameraX 获取实时相机预览帧，计算实时灰度直方图和图像质量指标，并基于本地规则生成拍摄建议。用户可以点击“一键自动优化”让系统在设备支持范围内调整曝光补偿和补光灯，也可以继续手动微调。

### 5.2 功能边界

| 项目 | 第一版是否实现 | 说明 |
|---|---|---|
| 实时相机预览 | 是 | 使用 CameraX Preview |
| 实时帧直方图 | 是 | 使用 ImageAnalysis |
| 智能判断当前画面 | 是 | 本地规则生成建议 |
| 一键自动优化 | 是 | 控制曝光补偿和补光灯 |
| 手动曝光调节 | 是 | 曝光 + / 曝光 - |
| 手动补光灯开关 | 是 | 开启/关闭 Torch |
| 拍照 | 建议实现 | 使用 ImageCapture |
| 变焦 | 否 | 暂不实现 |
| 对焦 | 否 | 暂不实现 |
| 白平衡 / ISO / 快门 | 否 | 暂不实现 |
| 真实 AI API | 否 | 第一版不联网，仅预留接口 |

### 5.3 页面交互

```text
首页
  ↓
点击“智能拍摄助手”
  ↓
进入相机页面
  ↓
显示实时预览和实时直方图
  ↓
用户点击“智能判断当前画面”
  ↓
系统显示拍摄建议，但不修改相机参数
  ↓
用户点击“一键自动优化”
  ↓
系统自动调整曝光补偿和补光灯
  ↓
用户可继续手动点击“曝光 + / 曝光 - / 补光灯开关”
  ↓
实时直方图持续反馈画面变化
  ↓
用户点击“拍照”
```

### 5.4 CameraX 技术路线

```text
CameraX Preview
    用于显示实时相机画面

CameraX ImageAnalysis
    用于获取实时预览帧
    每 300ms–500ms 分析一帧
    使用 KEEP_ONLY_LATEST，避免分析阻塞预览

CameraX ImageCapture
    用于拍照保存当前画面

CameraControl
    用于调整曝光补偿和补光灯

CameraInfo
    用于查询曝光补偿范围、当前曝光值、是否支持闪光灯
```

### 5.5 实时分析策略

| 项目 | 策略 |
|---|---|
| 分析频率 | 每 300ms–500ms 分析一次 |
| 分析分辨率 | 优先 640×480 或 1280×720 |
| 背压策略 | KEEP_ONLY_LATEST |
| 帧处理 | Analyzer 完成后必须关闭 ImageProxy |
| UI 更新 | 只发布最新一次分析结果 |
| 算法复用 | 复用灰度直方图与质量分析模块 |
| 性能目标 | 不影响相机预览流畅性 |

### 5.6 智能拍摄建议规则

输入指标：

| 指标 | 说明 |
|---|---|
| `averageGray` | 平均灰度 |
| `darkRatio` | 暗部占比 |
| `brightRatio` | 亮部占比 |
| `shadowClippingRatio` | 暗部溢出占比 |
| `highlightClippingRatio` | 高光溢出占比 |
| `grayStdDev` | 灰度标准差 |
| `currentExposureIndex` | 当前曝光补偿 |
| `exposureRange` | 设备支持曝光范围 |
| `torchAvailable` | 是否支持补光灯 |
| `torchEnabled` | 补光灯是否开启 |

输出结果：

```kotlin
data class PhotoCoachResult(
    val sceneStatus: PhotoSceneStatus,
    val reason: String,
    val advice: String,
    val exposureDelta: Int,
    val torchAction: TorchAction
)

enum class TorchAction {
    KEEP,
    TURN_ON,
    TURN_OFF
}
```

规则表：

| 状态 | 判断条件 | 曝光建议 |
|---|---|---:|
| 严重欠曝 | `averageGray < 60` 或 `shadowClippingRatio > 0.20` | `+2` |
| 中度偏暗 | `averageGray < 85` 或 `darkRatio > 0.60` | `+1` |
| 轻微偏暗 | `averageGray < 100` 且 `darkRatio > 0.45` | `+1` |
| 严重过曝 | `averageGray > 205` 或 `highlightClippingRatio > 0.18` | `-2` |
| 中度偏亮 | `averageGray > 180` 或 `brightRatio > 0.60` | `-1` |
| 轻微偏亮 | `averageGray > 165` 且 `brightRatio > 0.45` | `-1` |
| 低对比度 | `grayStdDev < 35` 且亮度无明显异常 | `0` |
| 正常 | 亮度和溢出均较正常 | `0` |

补光灯规则：

| 情况 | 补光灯建议 |
|---|---|
| 严重欠曝且设备支持补光灯 | TURN_ON |
| 严重欠曝且曝光补偿接近上限 | 优先 TURN_ON |
| 中度偏亮或严重过曝且补光灯已开启 | TURN_OFF |
| 其他情况 | KEEP |

### 5.7 自动优化逻辑

```text
读取最近一次实时直方图指标
  ↓
RuleBasedPhotoCoach 生成建议
  ↓
计算目标曝光补偿 = 当前曝光补偿 + exposureDelta
  ↓
将目标曝光补偿限制在设备支持范围内
  ↓
调用 CameraControl.setExposureCompensationIndex(target)
  ↓
根据 torchAction 调用 enableTorch(true/false)
  ↓
展示实际执行结果
```

### 5.8 用户手动微调逻辑

| 按钮 | 行为 |
|---|---|
| 曝光 + | 当前曝光补偿 +1，并限制到上限 |
| 曝光 - | 当前曝光补偿 -1，并限制到下限 |
| 补光灯开关 | 如果支持闪光灯，则切换补光灯状态 |
| 智能判断当前画面 | 只生成建议，不修改参数 |
| 一键自动优化 | 根据建议执行曝光和补光灯控制 |

用户手动微调后，系统不再自动覆盖用户选择，除非用户再次点击“一键自动优化”。

## 六、功能三：直方图应用分析增强

### 6.1 功能目标

增强静态图片和 ROI 区域分析的解释能力。系统在现有灰度直方图基础上增加 RGB 三通道统计，并基于亮度、曝光、对比度和颜色偏移生成自然语言图像质量判断。

第一版不接联网 API，所有自然语言判断由本地规则生成。

### 6.2 新增指标

| 指标 | 说明 |
|---|---|
| `redCounts[256]` | R 通道频次 |
| `greenCounts[256]` | G 通道频次 |
| `blueCounts[256]` | B 通道频次 |
| `avgRed` | R 通道均值 |
| `avgGreen` | G 通道均值 |
| `avgBlue` | B 通道均值 |
| `dominantChannel` | 主导通道 |
| `channelImbalance` | 通道差异度 |
| `colorCastStatus` | 偏红、偏绿、偏蓝或较均衡 |
| `effectiveGrayRange` | 有效灰度范围 |
| `shadowClippingRatio` | 暗部溢出 |
| `highlightClippingRatio` | 高光溢出 |

### 6.3 RGB 偏色判断规则

| 条件 | 判断 |
|---|---|
| 三通道均值最大差异 `< 15` | 色彩较均衡 |
| R 明显高于 G、B，且差异 `>= 20` | 画面可能偏红或偏暖 |
| G 明显高于 R、B，且差异 `>= 20` | 画面可能偏绿 |
| B 明显高于 R、G，且差异 `>= 20` | 画面可能偏蓝或偏冷 |
| 差异在 `15–20` | 轻微偏色 |

### 6.4 自然语言判断结构

输出分为五部分：

| 字段 | 内容 |
|---|---|
| `brightnessDescription` | 亮度描述 |
| `exposureDescription` | 曝光描述 |
| `contrastDescription` | 对比度描述 |
| `colorDescription` | RGB 偏色描述 |
| `summary` | 综合质量判断 |
| `advice` | 处理建议 |

示例：

```text
当前图像整体偏暗，暗部像素占比较高，可能存在欠曝风险。灰度标准差偏低，图像层次变化不足。RGB 三通道中蓝色通道均值明显高于红色和绿色，画面可能存在轻微偏蓝现象。建议提高曝光补偿或改善光照条件，必要时可在后续处理中进行对比度增强。
```

### 6.7 验收标准

- 系统能够统计 R/G/B 三通道频次和均值。
- 系统能够判断图像是否存在明显偏红、偏绿或偏蓝。
- 系统能够生成中文自然语言质量判断。
- 自然语言判断内容与数值指标一致。
- 不联网、不调用 API、不上传图片。
- 静态全图和 ROI 区域都能展示 RGB 指标和自然语言判断。
- 原有灰度直方图、性能卡和两种算法不受影响。

| 任务编号 | 任务 | 优先级 | 状态 |

## 九、风险与降级方案

| 风险 | 影响 | 降级方案 |
|---|---|---|
| ROI 坐标映射复杂 | 框选区域与实际裁剪不一致 | 第一版仅支持在固定比例预览容器内框选，并增加边界校正 |
| ROI 裁剪增加内存 | 大图局部裁剪可能产生额外 Bitmap | 限制最小和最大 ROI 尺寸，必要时提示用户缩小区域 |
| CameraX 兼容问题 | 部分设备相机控制不可用 | 显示设备不支持提示，不影响静态图片功能 |
| 实时分析卡顿 | 相机预览不流畅 | 降低分析分辨率，延长分析间隔，使用 KEEP_ONLY_LATEST |
| 曝光补偿不支持 | 自动优化无法调曝光 | 保留智能建议和补光灯控制 |
| 设备无补光灯 | 无法开灯 | 禁用补光灯按钮并显示提示 |
| 自然语言判断误导 | 规则过于绝对 | 文案使用“可能”“建议”，避免专业诊断口吻 |
| 新功能破坏旧功能 | 影响原有主流程 | 每阶段完成后执行静态图片回归测试 |

## 十、最终展示口径

本轮迭代完成后，系统功能可描述为：

> 系统在原有静态图片直方图计算基础上，进一步扩展了直方图的应用场景。用户既可以对整张图片进行灰度直方图与 RGB 三通道分析，也可以自由框选 ROI 局部区域并将其作为新的分析对象重新计算直方图。系统基于直方图指标生成自然语言图像质量判断，解释图像是否存在偏暗、偏亮、过曝、欠曝、低对比度或偏色问题。同时，系统新增智能拍摄助手，通过实时相机帧直方图分析当前画面，并在设备支持范围内自动调节曝光补偿和补光灯，用户也可以继续手动微调。这样，系统从基础直方图生成工具扩展为面向图像质量分析和拍摄优化的移动端应用。
