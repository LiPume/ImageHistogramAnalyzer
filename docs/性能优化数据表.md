# 性能优化数据表

## 一、统计口径

- 核心耗时：从像素访问开始，到 256-bin 频次完成 0–100 归一化为止；图片解码和 Compose Canvas 绘制不计入。
- 开发基准：Debug/API 36 模拟器，先进行 20 次全局 JIT 预热，再对每个尺寸预热 5 次、正式运行 10 次。
- 主指标：中位数；同时保留 P90 和最差值，不能用单次最快值作结论。
- 正确性门禁：同一输入的 Kotlin v2、Native v3、两种课程方案必须逐 bin 一致，才允许比较速度。
- 模拟器只有 1 个虚拟 CPU；它适合验证 Native 直读和 C++ 热循环，但不能证明手机八核的并行加速。

## 四、真机版本迭代表

固定设备：Xiaomi 14，第三代骁龙 8，16GB RAM，Android 16。v1/v2 来自同分辨率、同最高频次的杯子场景截图；v3 开始应固定原图并记录 SHA-256。

| 版本 | 主要改进 | 优先灰度化 | 统计时灰度化 | 相对上一版提升 | 300ms 状态 |
|---|---|---:|---:|---|---|
| v1.0 | MVP 两方案、基础分段前总计时 | 1415.321ms | 1304.234ms | 基线 | 未达到 |
| v2.0 | Kotlin 索引 `while`、分块取消、整数定点快路径 | 775.583ms | 673.735ms | 1.82× / 1.94×；下降 45.2% / 48.3% | 未达到 |
| v3.0 | NDK 28.2；`AndroidBitmap_lockPixels`；O3 C++；线程私有 256-bin；1-byte 灰度缓冲；8 线程；Kotlin 回退 | 56.957ms（单次） | 22.244ms（单次） | 参考约 13.62× / 30.29×；v2/v3 非严格同文件 | 已达到 |
| v4.0 验收轮次 | 固定 light UI、质量/预算可视化与错误恢复；核心仍为 Native v3.0 | 38.291ms（单次） | 23.763ms（单次） | 无算法变更，不把单次波动表述为新加速 | 已达到 |

v3.0 截图使用 `1783408658743.jpg`，3072×4096、12,582,912 像素；图片解码 117.465ms，两个方案最高频次均为 124,455。优先灰度化分段为像素访问 0.002ms、灰度转换 40.850ms、统计 8.916ms、归一化 0.722ms、其他 6.467ms、合并 0.001ms；统计时灰度化分段为像素访问 0.003ms、融合 21.264ms、归一化 0.590ms、其他 0.385ms、合并 0.002ms。

v4.0 是 UI 与基础功能验收轮次，性能卡片仍明确显示 `Native 多线程 v3.0` 和 8 个工作线程。用户在 Xiaomi 14 对同一尺寸 3072×4096 图片取得：解码 127.882ms；优先灰度化核心 38.291ms（灰度 20.225ms、统计 8.446ms），统计时灰度化核心 23.763ms（融合 22.072ms）；两方案最高频次同为 124,455。该轮证明新 UI 没有破坏算法链路和 300ms 结论，不代表 v4 新增了算法加速。

注意：v2 截图最高频次为 121,520，v3 为 124,455，说明并非严格同一文件，因此版本加速倍数只作方向参考。课程最终报告前仍应固定 v3 原图，每方案预热 5 次、正式运行至少 10 次，补充中位数、P90 和最差值；这属于验收数据整理，不再触发新一轮性能开发。

## 五、优化技术分析与决策

| 方法 | 本轮是否采用 | 分析与边界 |
|---|---|---|
| 分支/定点快路径 | 已采用 | 非 `.5` 灰度直接走整数路径；仅 16,782 个精确边界回退浮点，保持课程公式语义。 |
| 多线程分块 | 已采用并真机运行 | 按行分块，每线程独立 256-bin，结束后归并，避免共享计数器竞争；Xiaomi 14 显示 8 个工作线程。单次已达标，不再单独调参。 |
| NDK Bitmap 直读 | 已采用 | 使用稳定的 `AndroidBitmap_lockPixels`/`unlockPixels`，取消约 48MiB Java `IntArray` 彩色像素复制。 |
| C++ 编译优化 | 已采用 | C++20、`-O3`；禁用 `fast-math`，避免灰度取整语义漂移。编译器可能自动向量化，但本项目不把它表述为“已实现显式 SIMD”。 |
| ARM NEON 显式 SIMD | 已搁置 | 当前最慢方案单次 56.957ms，仅占预算约 19%；除非最终多轮 P90 超过 300ms，否则不实施。 |
| OpenCV/其他第三方库 | 已搁置 | 课程允许使用，但 Bitmap→Mat 转换、APK 体积和 ABI 维护都有成本；当前自研 Native 已有充足余量。 |

完整机器可读数据见 `docs/performance/模拟器性能测试_v3.csv`。
