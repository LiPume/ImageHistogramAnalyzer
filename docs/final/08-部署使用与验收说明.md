# 部署、使用与验收说明

## 一、开发环境

- Android Studio 最新稳定版；
- Android Studio 内置 JDK 21（项目 Gradle daemon toolchain 固定为 Java 21；源码字节码目标为 Java 11）；
- Android SDK 36 / API 36.1；
- Android NDK `28.2.13676358`；
- 一台 API 23+ 模拟器或 Android 手机。

项目不提交 `local.properties`，Windows 和 macOS 组员首次打开时由 Android Studio 自动生成本机 SDK 路径。NDK 版本写在 Gradle 中，不依赖开发者的绝对路径。

## 二、拉取与构建

```bash
git clone git@github.com:LiPume/ImageHistogramAnalyzer.git
cd ImageHistogramAnalyzer
git pull origin main
```

macOS/Linux：

```bash
./gradlew :app:assembleDebug
```

Windows PowerShell/CMD：

```powershell
.\gradlew.bat :app:assembleDebug
```

Debug APK：`app/build/outputs/apk/debug/app-debug.apk`。

若提示缺少 NDK，在 Android Studio 的 SDK Manager → SDK Tools 中勾选 “NDK (Side by side)”，安装 `28.2.13676358`。不要修改项目 NDK 版本来迎合单台电脑。

## 三、自动化验证

macOS/Linux：

```bash
./gradlew :app:testDebugUnitTest :app:assembleDebug :app:assembleDebugAndroidTest :app:lintDebug
./gradlew :app:connectedDebugAndroidTest
```

Windows 把 `./gradlew` 换成 `.\gradlew.bat`。第二条需要已连接并授权的模拟器或手机；显式性能 benchmark 在普通测试中会按设计跳过。

## 四、安装

```bash
adb devices
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

也可以直接在 Android Studio 选择设备后点击 Run。正式 Release APK 使用 Android Studio 的 Generate Signed APK 生成，签名文件不得提交仓库。

准备并安装完整测试图片集：

```bash
./scripts/prepare_final_test_images.sh
./scripts/install_test_images.sh test/final
```

第一条命令在 macOS/Xcode 环境生成精确 PNG 并下载公开 WebP；第二条把图片递归推送到手机/模拟器并触发 MediaStore 扫描。Windows 组员可从已经准备好的私有测试目录复制图片，或参照 [测试图片清单](../测试图片清单.md) 核对哈希。

## 五、用户操作

静态图片分析：

1. 启动 App，在首页点击“选择图片”；
2. 在系统选择器中选择图片；Xiaomi 14 可从选择器直接拍照；
3. 核对预览、文件名、分辨率、像素数和类型；
4. 向下滚动，选择“优先灰度化”或“统计时灰度化”；
5. 点击“计算并绘制直方图”；
6. 查看白底黑柱直方图、质量分析和性能卡片；
7. 同图切换另一方案并重新计算；
8. 如需局部分析，点击“选择局部分析区域”，在图片上拖动矩形框，点击“确认区域并重新分析”；结果页会显示局部区域尺寸、像素数、占比、局部直方图、RGB 和质量判断；
9. 点击“恢复全图分析”可回到原图并重新生成全图结果；
10. 点击“重新选择图片”测试换图；
11. 点击“返回首页”或系统返回键；首页可“继续查看分析”。

实时相机分析：

1. 首页点击“打开实时拍摄分析”；
2. 首次进入时点击“授权相机并开始预览”，在系统弹窗中允许相机权限；
3. 查看相机预览，等待约 1 秒出现“实时亮度直方图”和“实时质量指标”；
4. 点击“智能判断当前画面”，查看画面状态、判断依据、曝光建议、补光灯建议和拍摄建议；
5. 将镜头对准暗处、亮处和普通场景，观察直方图、平均亮度、暗部/亮部占比和质量分类是否随画面变化；需要刷新建议时再次点击“智能判断当前画面”；
6. 点击“返回首页”或系统返回键，确认可以离开实时页且静态图片分析入口仍可用。

说明：实时相机页当前使用 CameraX `YUV_420_888` 的 Y 通道作为亮度信号，每约 400ms 分析一帧，目的是保持预览流畅。静态图片课程验收仍使用指定 RGB 灰度公式和 Native v3 两方案。智能建议只生成文字建议，不会自动修改相机参数。

## 六、界面结果解释

- 横轴 0–255：从黑到白的灰度；
- 纵轴 0–100：相对最高频次归一化高度，不是像素百分比；
- 最高灰度频次：出现次数最多的灰度 bin 的像素数；
- 暗部/亮部：分别为灰度 0–63、192–255 的占比；
- 300ms 条：核心计算占预算比例，解码和 Canvas 不计入核心指标；
- `Native 多线程 v3.0`：当前优化主路径；Native 失败时自动回退 Kotlin。

## 七、常见问题

| 问题 | 处理 |
|---|---|
| 选图后没有自动直方图 | 这是预期行为；先选择方案，再点击计算按钮 |
| 页面下方看不到 | 页面可纵向滚动，继续向上滑动内容 |
| 局部区域确认按钮不可点 | 需先选择一种计算方案，并在图片上拖出有效矩形 |
| 局部区域太小 | 系统会提示重新框选更大的区域 |
| 图片超过上限 | 选择不超过 1600 万像素的原图；系统不会静默缩小分析 |
| 损坏或无权限 | 按中文提示重新选择；已有成功结果会保留 |
| Windows 缺少 NDK | SDK Manager 安装固定的 side-by-side NDK 版本 |
| 系统选择器没有拍照 | 取决于设备实现，可先用系统相机拍照再从相册选择 |
| 实时相机没有预览 | 确认已授予相机权限；模拟器需启用虚拟摄像头；真机可返回首页后重新进入 |
| 实时相机和静态图结果略有差异 | 实时页第一版使用 Y 通道亮度，静态图使用课程 RGB 灰度公式，二者口径已在界面和文档中区分 |
| 实时页没有一键曝光/补光灯 | 当前只实现文字建议；真实曝光、补光灯控制和拍照属于后续 CAM-08~13 |
| connected test 失败 | 确认 `adb devices` 显示 `device`，并关闭多个设备或设置 `ANDROID_SERIAL` |

正式真机性能步骤见 [真机测试与性能验收记录](07-真机测试与性能验收记录.md)。

## 八、Windows 组员注意事项

- Git 只同步 NDK/C++ 源码和固定版本号，不同步 macOS 的 NDK 安装目录；
- 首次 Gradle Sync 会提示安装 `28.2.13676358`，由 Android Studio SDK Manager 安装即可；
- 不要提交 `local.properties`，也不要把 `sdk.dir` 或 `ndk.dir` 写入 Gradle；
- 若公司/校园网络无法下载依赖，可让 Android Studio 使用已有 SDK 缓存，但不要修改项目版本来绕过；
- PowerShell 脚本命令使用 `./gradlew` 对应的 `\.\gradlew.bat`。

## 九、最终封版步骤

1. 在最终 commit 上重新运行四项 Gradle 门禁和连接设备测试；
2. 生成最终 APK，记录构建类型、文件大小和 SHA-256；
3. 在 Xiaomi 14 安装并完成 17-ID 表格；
4. 复核实时相机页的授权、预览、实时直方图、智能判断和返回首页；
5. 将 L1 数值、实时页截图、commit/tag 和提交日期回填文档与 PPT；
6. 检查 Git 不包含测试原图、签名材料、构建目录或本机配置；
7. 创建阶段性 commit 和课程提交 tag。
