# 测试计划与测试用例

## 一、测试目标

验证 App 可安装运行、图片链路可靠、两种算法正确一致、质量指标可复算、UI 状态清晰、错误可恢复，并在 Xiaomi 14 上完成 17 个图片 ID 的逐图验收，以 L1 验证核心计算 300ms 目标。

## 二、测试层级

| 层级 | 重点 | 工具/命令 |
|---|---|---|
| JVM 单元测试 | 灰度、归一化、两方案一致、质量阈值、计时模型 | `./gradlew :app:testDebugUnitTest` |
| Android 仪器测试 | Native/JNI、Bitmap、ViewModel、Photo Picker 回调、ROI 裁剪/恢复 | `./gradlew :app:connectedDebugAndroidTest` |
| Compose UI 测试 | 首页、导航、状态、按钮、滚动、图表语义、ROI 拖拽确认、实时相机权限/结果状态/智能建议 | 同上 |
| 静态与构建 | Debug APK、测试 APK、Lint、四 ABI Native | `assembleDebug`、`assembleDebugAndroidTest`、`lintDebug` |
| 真机验收 | 实际选图/拍照、连续换图、两方案性能、电量与截图 | 文档 07 手工执行 |

最近一次自动化结果（2026-07-09，CameraX v5.0-f 工作区）：`./gradlew :app:testDebugUnitTest` 通过，其中 JVM 58 项通过；`./gradlew :app:assembleDebug :app:assembleDebugAndroidTest :app:lintDebug` 通过；`ANDROID_SERIAL=emulator-5554 ./gradlew :app:connectedDebugAndroidTest` 在 API 36.1 AVD 上 31 项运行、1 项显式性能基准按设计跳过、0 失败。最终提交前仍需在最终 commit 上复跑一次。

## 四、算法正确性用例

| TC-A07 | `.5` 边界颜色 | Native 与 Kotlin 逐 bin 相等 |

## 六、UI 与无障碍用例

- 主按钮文字必须与立即动作一致；
- 加载时显示进度并阻止重复操作；
- 错误时有标题、原因和重新选择入口；
- 直方图和比例条提供可读语义，精确数字保留；
- 固定 light 在系统 dark mode 下仍可读；
- 1.5× 字体、窄屏和 edge-to-edge 不遮挡；
- 可交互区域至少 48dp。
