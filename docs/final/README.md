# 最终开发文档总入口

本目录是课程提交与答辩使用的最终文档集。现有 `docs/` 根目录文件保留开发过程、调研和原始测试证据；最终提交时优先从本页进入，不需要在多个历史文档中寻找结论。

推荐课堂验收演示稿：[第十四组 ImageHistogramAnalyzer 课堂汇报终版](第十四组_ImageHistogramAnalyzer_课堂汇报.pptx)，共 15 页，使用 `test/v5.0/` 真机截图，包含课程目标、产品双路径、三层架构、双算法、性能优化、ROI、RGB、自然语言分析、CameraX 智能拍摄和测试保障。固定 L1 的最终真机数据尚未产生，PPT 中的历史数据只用于说明优化过程；最终 300ms 结论仍以 Xiaomi 14 的固定 L1 表为准。本次封版已获授权将 PPTX 和 XLSX 纳入 Git。

`11.pptx` 和 `12.pptx` 作为本轮 15 页/17 页编辑过程稿一并留档，正式汇报只使用上述带项目全名的终版文件。

## 三、封版规则

1. `[待填写]` 表示必须由最终真机、APK 或课程信息补充，不能猜测。
2. 最终课程验收采用 17 个 ID 各一行、两方案各运行一次；历史多轮模拟器数据只作为稳定性和优化方向证据，不与本次逐图表混写。
3. 图片解码、核心计算和界面绘制分开描述；300ms 指标对应核心计算。
4. 两种方案的 256-bin 结果不一致时，性能结论无效。
5. 最终提交前填写版本、commit、APK SHA-256，并执行文档 09 的交付清单。
6. PPTX 和 XLSX 已在本次封版中纳入 Git；后续修改二进制文件时必须说明用途，Office 的 `~$` 临时锁文件仍禁止提交。
