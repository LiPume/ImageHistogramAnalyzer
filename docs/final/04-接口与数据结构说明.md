# 接口与数据结构说明

## 一、接口范围

本项目是纯本地 Android 应用，没有 HTTP API、数据库表或云端接口。本文件中的“接口”指模块之间的 Kotlin/JNI 契约、UI 事件和结果数据结构。

## 二、图片读取接口

| 接口 | 输入 | 输出 | 约束/异常 |
|---|---|---|---|
| `ImageLoader.load(uri)` | `Uri` | `DecodedImage` | 挂起函数；生产实现切换到 IO 调度器 |
| `BitmapDecoder.decode(uri)` | Content URI | 软件 `Bitmap` + `ImageMetadata` | 仅通过 `ContentResolver`；最大 1600 万像素 |
| `BitmapPixelReader.read(bitmap)` | 软件 Bitmap | ARGB `IntArray` | Kotlin 回退路径使用 |

图片异常：`ImageTooLargeException`、`ImageOpenException`、`ImageDecodeException`、`UnsupportedImageTypeException` 和 `SecurityException`。ViewModel 将其转换为中文可恢复提示。

## 三、直方图算法接口

```kotlin
interface HistogramCalculator {
    val strategy: HistogramCalculationStrategy
    fun calculateMeasured(
        pixels: IntArray,
        cancellationCheck: () -> Unit = {},
    ): MeasuredHistogramResult
}
```

策略枚举：

- `PRE_GRAYSCALE`：优先灰度化；
- `GRAYSCALE_WHILE_COUNTING`：统计时灰度化。

输入必须是完整图片的 ARGB 像素。输出必须包含 256 个频次、256 个 `0..100` 高度、像素总数和最大频次。

## 四、Native/JNI 接口

`NativeHistogramBridge.calculate(bitmap, strategy, workerCount, timings)` 返回 256 项 `IntArray`。`strategy` 中 0 表示优先灰度化，1 表示统计时灰度化。

`timings` 至少包含 6 项：

| 索引 | 内容 |
|---:|---|
| 0 | Bitmap 锁定/像素访问耗时 |
| 1 | 灰度转换耗时；融合方案为 0 |
| 2 | 频次统计或融合阶段耗时 |
| 3 | 多线程结果合并耗时 |
| 4 | Native 内部总耗时 |
| 5 | 实际工作线程数 |

Native 加载失败时 `NativeBitmapHistogramCalculator.isAvailable=false`，ViewModel 自动使用 Kotlin 实现。

## 五、核心数据结构

| 类型 | 关键字段 | 说明 |
|---|---|---|
| `ImageMetadata` | 名称、MIME、宽、高、像素数 | UI 不直接查询 ContentResolver |
| `HistogramResult` | `counts[256]`、`normalizedHeights[256]`、pixelCount、maxCount | 构造时复制数组，防止外部修改 |
| `HistogramPerformanceMetrics` | 像素、灰度、计数、归一化、合并、核心总计、引擎、线程 | 纳秒保存，UI 转毫秒 |
| `ImageQualityResult` | 平均灰度、暗/亮占比、标准差、分类 | 只由直方图计算 |
| `AnalyzerUiState` | 加载、图片、策略、结果、错误 | 页面唯一状态源 |

## 六、ViewModel 操作接口

| 方法 | 前置条件 | 状态变化 |
|---|---|---|
| `selectImage(uri)` | 系统选择器返回非空 URI | 取消旧任务，解码；成功重置分析状态，失败保留旧成功内容 |
| `selectStrategy(strategy)` | 已有图片且未处理 | 设置策略并清除旧直方图/质量/性能结果 |
| `calculateHistogram()` | 已有图片和策略 | 后台计算，成功发布直方图、质量和性能结果 |

## 七、Compose 事件接口

- 首页：`onPickImage`、`onResumeAnalysis`；
- 分析页：`onBackHome`、`onPickImage`、`onSelectStrategy`、`onCalculate`；
- `ImageHistogramContent` 只根据 `AppDestination` 和 `AnalyzerUiState` 渲染，不直接执行业务算法。

## 八、一致性与版本约束

- Kotlin、Native 和两种策略必须共享相同公式、取整、Alpha 和归一化规则；
- `HistogramResult.counts.sum()` 必须等于 `pixelCount`；
- 只有逐 bin 相等时才能报告加速；
- 接口字段或计时口径变化时，必须同步测试报告、性能表和本文件。
