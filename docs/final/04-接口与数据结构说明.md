# 接口与数据结构说明

## 一、接口范围

本项目是纯本地 Android 应用，没有 HTTP API、数据库表或云端接口。本文件中的“接口”指模块之间的 Kotlin/JNI 契约、UI 事件和结果数据结构。

## 二、图片读取接口

| 接口 | 输入 | 输出 | 约束/异常 |
|---|---|---|---|
| `ImageLoader.load(uri)` | `Uri` | `DecodedImage` | 挂起函数；生产实现切换到 IO 调度器 |
| `BitmapDecoder.decode(uri)` | Content URI | 软件 `Bitmap` + `ImageMetadata` | 仅通过 `ContentResolver`；最大 1600 万像素 |
| `BitmapPixelReader.read(bitmap)` | 软件 Bitmap | ARGB `IntArray` | Kotlin 回退路径使用 |
| `PreviewRoiMapper.mapToBitmap(rect, layout)` | 预览矩形 + `ContentScale.Fit` 布局 | `RoiSelection` | 处理缩放、居中、留白、反向拖拽和过小区域 |
| `RoiBitmapCropper.crop(bitmap, roi)` | 当前 Bitmap + Bitmap 坐标 ROI | 裁剪后的 Bitmap | 校验尺寸一致、边界和最小区域 |

图片异常：`ImageTooLargeException`、`ImageOpenException`、`ImageDecodeException`、`UnsupportedImageTypeException` 和 `SecurityException`。ViewModel 将其转换为中文可恢复提示。

## 三、实时相机接口

实时相机链路只服务于预览期的“亮度趋势观察”，不替代静态图片课程验收的 RGB 灰度公式。

| 接口 | 输入 | 输出 | 约束/异常 |
|---|---|---|---|
| `CameraScreen` | `CameraUiState` + 回调 | CameraX 权限、预览、实时直方图和质量指标 UI | 使用注入 `previewContent` 支持 Compose 测试不启动真实相机 |
| `RealtimeFrameAnalyzer.analyze(image)` | CameraX `ImageProxy` | `RealtimeCameraAnalysis` 回调 | `KEEP_ONLY_LATEST`；默认约 400ms 分析一帧；`finally` 中关闭 `ImageProxy` |
| `RealtimeLumaHistogramAnalyzer.analyze(...)` | Y 通道 `ByteArray`、宽高、`rowStride`、`pixelStride` | 256-bin 亮度直方图 + 质量指标 | 校验尺寸、stride 和数据长度；纯 Kotlin，可 JVM 测试 |
| `CameraViewModel` | 权限、绑定、帧结果、错误事件 | `CameraUiState` | 只保存最新帧结果和错误信息 |
| `RuleBasedPhotoCoach.analyze(analysis)` | 最近一帧 `RealtimeCameraAnalysis` | `PhotoCoachResult` | 只生成本地建议，不调用 CameraControl |

当前实时页使用 CameraX `YUV_420_888` 的 Y 通道作为亮度信号。这样避免每帧做完整 YUV→RGB 转换导致预览卡顿；后续如要把实时页改成完整 RGB 灰度公式，需要新增转换成本计时和真机流畅性测试。

## 四、直方图算法接口

```kotlin
interface HistogramCalculator {
    val strategy: HistogramCalculationStrategy
    fun calculateMeasured(
        pixels: IntArray,
        cancellationCheck: () -> Unit = {},
    ): MeasuredHistogramResult
}
```

策略枚举：

- `PRE_GRAYSCALE`：优先灰度化；
- `GRAYSCALE_WHILE_COUNTING`：统计时灰度化。

输入必须是完整图片的 ARGB 像素。输出必须包含 256 个频次、256 个 `0..100` 高度、像素总数和最大频次。

## 五、Native/JNI 接口

`NativeHistogramBridge.calculate(bitmap, strategy, workerCount, timings)` 返回 256 项 `IntArray`。`strategy` 中 0 表示优先灰度化，1 表示统计时灰度化。

`timings` 至少包含 6 项：

| 索引 | 内容 |
|---:|---|
| 0 | Bitmap 锁定/像素访问耗时 |
| 1 | 灰度转换耗时；融合方案为 0 |
| 2 | 频次统计或融合阶段耗时 |
| 3 | 多线程结果合并耗时 |
| 4 | Native 内部总耗时 |
| 5 | 实际工作线程数 |

Native 加载失败时 `NativeBitmapHistogramCalculator.isAvailable=false`，ViewModel 自动使用 Kotlin 实现。

## 六、核心数据结构

| 类型 | 关键字段 | 说明 |
|---|---|---|
| `ImageMetadata` | 名称、MIME、宽、高、像素数 | UI 不直接查询 ContentResolver |
| `HistogramResult` | `counts[256]`、`normalizedHeights[256]`、pixelCount、maxCount | 构造时复制数组，防止外部修改 |
| `HistogramPerformanceMetrics` | 像素、灰度、计数、归一化、合并、核心总计、引擎、线程 | 纳秒保存，UI 转毫秒 |
| `ImageQualityResult` | 平均灰度、暗/亮占比、标准差、分类 | 只由直方图计算 |
| `RoiSelection` | left/top/width/height、原图宽高、像素数、面积占比 | Bitmap 坐标系中的局部区域 |
| `AnalysisTargetInfo` | 全图/局部区域、宽高、像素数、占比 | UI 展示当前分析对象，不持有 Bitmap |
| `AnalyzerUiState` | 加载、图片、策略、结果、错误 | 页面唯一状态源 |
| `RealtimeCameraAnalysis` | 亮度直方图、质量结果、帧宽高、时间戳、来源 | 实时相机页最新一帧分析结果 |
| `PhotoCoachResult` | 场景状态、判断依据、建议、曝光增量、补光灯动作 | 实时智能拍摄建议，不直接执行硬件控制 |
| `CameraUiState` | 权限、绑定状态、最新实时结果、智能建议、错误 | 实时相机页面唯一状态源 |

## 七、ViewModel 操作接口

| 方法 | 前置条件 | 状态变化 |
|---|---|---|
| `selectImage(uri)` | 系统选择器返回非空 URI | 取消旧任务，解码；成功重置分析状态，失败保留旧成功内容 |
| `selectStrategy(strategy)` | 已有图片且未处理 | 设置策略并清除旧直方图/质量/性能结果 |
| `calculateHistogram()` | 已有图片和策略 | 后台计算，成功发布直方图、质量和性能结果 |
| `startRoiSelection()` | 已有图片且未处理 | 进入预览框选模式，不改变已有结果 |
| `confirmRoiSelection(rect, layout)` | 已有图片、策略和有效预览矩形 | 映射并裁剪 ROI，复用现有分析链路生成局部结果 |
| `restoreFullImageAnalysis()` | 已经保留原图 Bitmap | 恢复原图元数据并重新生成全图结果 |

`CameraViewModel`：

| 方法 | 前置条件 | 状态变化 |
|---|---|---|
| `setCameraPermission(granted)` | 系统权限状态变化 | 更新权限状态；授权成功时清理旧错误 |
| `setBindingCamera(binding)` | CameraX 绑定开始或结束 | 更新加载/绑定状态 |
| `onFrameAnalyzed(result)` | ImageAnalysis 产生新结果 | 保存最新实时直方图和质量指标，清除错误 |
| `judgeCurrentFrame()` | 已有最新实时帧 | 基于最近一帧生成 `PhotoCoachResult` |
| `onCameraError(message)` | CameraX 绑定或帧分析失败 | 停止加载并显示中文错误 |

## 八、Compose 事件接口

- 首页：`onPickImage`、`onResumeAnalysis`、`onOpenCamera`；
- 分析页：`onBackHome`、`onPickImage`、`onSelectStrategy`、`onCalculate`、`onStartRoiSelection`、`onCancelRoiSelection`、`onConfirmRoiSelection`、`onRestoreFullImage`；
- 实时相机页：`onBackHome`、`onRequestPermission`、`onCameraBindingChanged`、`onFrameAnalyzed`、`onCameraError`、`onJudgeCurrentFrame`；
- `ImageHistogramContent` 只根据 `AppDestination`、`AnalyzerUiState` 和 `CameraUiState` 渲染，不直接执行业务算法。

## 九、一致性与版本约束

- Kotlin、Native 和两种策略必须共享相同公式、取整、Alpha 和归一化规则；
- `HistogramResult.counts.sum()` 必须等于 `pixelCount`；
- 只有逐 bin 相等时才能报告加速；
- 实时相机 Y 通道亮度直方图必须在 UI 和文档中明确标注为实时预览口径，不得冒充静态 RGB 公式验收结果；
- 接口字段或计时口径变化时，必须同步测试报告、性能表和本文件。

## 十一、字段单位与边界

- 所有内部阶段耗时以纳秒 `Long` 保存，界面显示时转换为毫秒；
- 图片宽高和频次使用整数，像素数校验使用 `Long` 防止乘法溢出；
- `counts` 和 `normalizedHeights` 长度必须为 256；
- Native 工作线程数限制在 `1..8`；
- 300ms 判定只读取 `coreTotalNanos`，不把解码和 Compose 绘制混入。
