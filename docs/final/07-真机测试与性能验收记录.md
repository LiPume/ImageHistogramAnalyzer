# 真机测试与性能验收记录

## 一、最终验收口径

本轮采用“一张图片 ID 一行”的课程验收方式：17 个 ID 各测试一次；除超限图 B02 外，同一行依次运行“优先灰度化”和“统计时灰度化”。不再要求同一 ID 重复 10 次，也不记录温度。

该口径用于完成全图片功能、正确性、兼容性和 300ms 验收，不用于宣称统计意义上的长期稳定性。历史模拟器多轮基准仍保留在 [性能优化数据表](../性能优化数据表.md)，作为 v3 优化方向和可重复性证据。

## 四、手机测试准备

1. 电量建议在 30% 以上，测试期间不要充电，并关闭省电模式；
2. 安装待验收 APK，确认性能卡显示 `Native 多线程 v3.0` 和 8 个工作线程；
3. 打开 Excel 或 CSV，每个 ID 只占一行；
4. 每个 ID 开始时填写当时电量；不需要测温度；
5. 选择原图，不使用界面截图或社交软件二次压缩图。

可选设备信息命令：

```bash
adb devices
adb shell getprop ro.product.model
adb shell getprop ro.soc.model
adb shell getprop ro.build.version.release
adb shell getprop ro.build.version.sdk
adb shell getprop ro.build.display.id
adb shell dumpsys battery
```

## 五、每个 ID 的操作步骤

对 S01–S08、L1、B01、T1–T5、W01：

1. 点击“选择图片”，选择该 ID 对应图片；
2. 核对预览、文件名、分辨率和像素数；
3. 记录图片解码耗时和当前电量；
4. 选择“优先灰度化”，点击计算一次，填写 A 方案各阶段耗时、核心总计和最高频次；
5. 不换图，选择“统计时灰度化”，点击计算一次，填写 B 方案各阶段耗时、核心总计和最高频次；
6. 填写实际质量结论、结果或异常；Excel 自动判断最高频次是否一致以及两方案是否低于 300ms；
7. L1、B01、T2 和 W01 建议各保存一张能看清性能卡的截图，作为大图、边界、真实图和格式兼容证据。

对 B02：选择图片后应直接出现超过 1600 万像素的错误提示，不运行算法；只填写解码/错误结果和电量，算法耗时保持空白，不要填 0。

## 六、每行记录字段

固定元数据已预填，lzx 只需填写：

- 电量和解码耗时；
- A 方案：像素访问、灰度转换、频次统计、归一化、其他开销、合并、核心总计、最高频次；
- B 方案：像素访问、融合统计、归一化、其他开销、合并、核心总计、最高频次；
- 实际质量、实际结果或异常。

“不适用”保持空白或填 `NA`，不能为了让表格好看填 0。

## 七、判定规则

| 检查项 | 通过条件 |
|---|---|
| 图片读取 | 文件名、分辨率、像素数与清单一致 |
| 两方案一致 | 同图 A/B 最高频次一致；自动化已额外证明 256-bin 逐项一致 |
| 质量结论 | 与表内预期一致；JPEG/WebP 少量解码差异只影响边缘 bin，不应改变整体结论 |
| 性能 | L1 两个方案本次核心总计均 `<300ms` |
| B01 | 16,000,000 像素允许分析 |
| B02 | 16,004,000 像素被拒绝，不进入核心计算 |
| 兼容性 | JPEG、PNG、WebP 均可选择和显示 |
