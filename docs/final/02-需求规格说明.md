# 需求规格说明

## 一、用户与使用场景

目标用户为课程验收教师和项目组成员。典型场景是在 Android 手机上选择或现场拍摄一张图片，查看灰度直方图、质量指标和计算耗时，并比较课程要求的两种处理流程。

## 三、算法规则

- 灰度公式固定为 `R×0.299 + G×0.587 + B×0.114`；
- 结果四舍五入到最近整数并限制到 `0..255`；
- Alpha 不改变 RGB 灰度统计语义，所有实现必须一致；
- 频次数组固定 256 项，频次总和等于像素数；
- 归一化公式为 `height[i] = round(count[i] / maxCount × 100)`，输出范围 `0..100`；
- 两种方案和 Kotlin/Native 实现必须逐 bin 相等。

## 四、质量判断规则

- 暗部：灰度 `0..63`；亮部：灰度 `192..255`；
- 平均灰度 `<85` 或暗部占比 `≥60%`：偏暗；
- 平均灰度 `>170` 或亮部占比 `≥60%`：偏亮；
- 其余情况下标准差 `<35`：低对比度；
- 其他：正常。

该结论用于课程演示和辅助观察，不属于专业图像质量诊断。

## 五、非功能需求

| 编号 | 类别 | 要求 |
|---|---|---|
| NFR-01 | 性能 | 正式 L1 真机验收中，两种方案本次核心计算均应低于 300ms；解码单列 |
| NFR-02 | 正确性 | 性能优化不得改变 256-bin、归一化高度或质量指标 |
| NFR-03 | 兼容性 | 支持 Android 6.0/API 23 及以上；最终重点设备为 Xiaomi 14/Android 16 |
| NFR-04 | 内存 | 单图不超过 1600 万像素；不静默降采样后冒充全图分析 |
| NFR-05 | 可用性 | 页面可完整滚动；按钮文案描述真实动作；触控区域至少 48dp |
| NFR-06 | 可访问性 | 重要图片、标题、直方图和比例条有可读语义，精确数值不只依赖颜色 |
| NFR-07 | 安全与隐私 | 全部本地处理；不上传图片；不提交个人照片、密钥、签名材料或本机配置 |
| NFR-08 | 可维护性 | UI、状态、图片读取、算法、Native 和文档按职责分离；`MainActivity` 保持轻量 |

## 六、限制与假设

- 系统选择器是否展示拍照入口由设备实现决定；Xiaomi 14 已验证可用；
- 超过 1600 万像素的图片会被拒绝；
- 正式性能结论待完成固定 L1 的两方案单次真机验收；历史多轮模拟器数据只作稳定性旁证；
- 当前不引入独立相机页、数据库、联网或第三方图像框架。

## 七、验收优先级

- P0：两种算法结果一致、L1 核心计算低于 300ms、B02 正确拒绝、App 无崩溃；
- P1：质量分类、JPEG/PNG/WebP、连续换图、返回首页、固定 light 和大字体可读；
- P2：Release 签名包、最终 tag、完整截图和答辩材料封版。
