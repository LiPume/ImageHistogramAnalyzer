# 图像直方图系统概要设计

## 一、当前版本

- 版本范围：阶段 1 MVP。
- 技术栈：Kotlin、Jetpack Compose、Material 3、ViewModel、StateFlow、协程。
- Android 配置：minSdk 23、targetSdk 36、compileSdk 36.1。
- 负责人：lzx。

## 二、模块关系

```text
系统 Photo Picker
       ↓ Uri
ImageRepository / BitmapDecoder
       ↓ DecodedImage(Bitmap + ImageMetadata)
AnalyzerViewModel
       ├─ BitmapPixelReader
       ├─ PreGrayscaleHistogramCalculator
       ├─ BaselineHistogramCalculator（统计时灰度化）
       ├─ HistogramNormalizer
       └─ AnalyzerUiState
              ↓
AnalyzerScreen / stateless components / HistogramCanvas
```

- `data/image`：通过 `ContentResolver` 读取 URI，校验尺寸并解码软件 Bitmap。
- `domain/model`：定义图片元数据和与 UI 解耦的直方图结果。
- `domain/histogram`：定义方案接口，执行两种灰度统计流程和 0..100 归一化。
- `ui/analyzer`：管理异步任务、取消、错误和页面状态。
- `ui/component`：只展示数据；Canvas 不读取 Bitmap、不执行统计。

## 三、核心规则

- 灰度公式：`gray = red * 0.299 + green * 0.587 + blue * 0.114`。
- 取整规则：四舍五入后限制到 `0..255`。
- Alpha 规则：Alpha 不参与公式，透明像素按解码后的 RGB 统计。
- 直方图：保存 256 个原始频次，并按最大频次等比例映射到 `0..100`。
- Canvas：使用 256×100 逻辑坐标，屏幕上等比缩放为白底黑柱。
- MVP 图片上限：1600 万像素；超过后显示错误，不静默缩小后冒充原图分析。

两种课程方案：

1. “优先灰度化”：彩色像素 → 完整灰度数组 → 频次统计 → 归一化 → Canvas 绘制。
2. “统计时灰度化”：彩色像素 → 遍历时灰度化并累计频次 → 归一化 → Canvas 绘制。

两种实现共用 `GrayscaleConverter` 与 `HistogramNormalizer`，确保公式、取整和归一化规则一致；自动化测试逐 bin 对比结果。选图只完成解码与预览，只有用户选择方案并点击计算按钮后才执行统计。

## 四、线程与生命周期

- URI 查询和 Bitmap 解码运行在 `Dispatchers.IO`。
- 批量取像素、灰度统计和归一化运行在 `Dispatchers.Default`。
- 每次重新选图会取消上一任务，并通过请求编号防止旧结果覆盖新状态。
- 大图计算每 16384 个像素检查一次协程取消，避免每像素检查带来明显开销。

## 五、性能口径

- 图片解码：从 URI 解码为 Bitmap，单独计时。
- 核心计算：`Bitmap.getPixels()`、灰度统计和 0..100 归一化，作为界面 300ms 判断指标。
- 当前界面结果是单次开发反馈；阶段 3 才进行预热、多轮、中位数和真机 Release 正式测试。
